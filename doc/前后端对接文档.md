## 二、后端API接口

### 基础信息
- **地址**: `http://localhost:8000`
- **文档**: `http://localhost:8000/docs` (Swagger UI)

### API列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/stats` | GET | 获取爬取状态统计 |
| `/api/notes` | GET | 获取笔记列表 |
| `/api/notes/{note_id}` | GET | 获取单个笔记详情 |
| `/api/notes/{note_id}/comments` | GET | 获取笔记的评论列表 |
| `/api/keywords` | GET | 获取所有关键词 |

---

### 1. 获取状态统计 `/api/stats`

**请求**: 无参数

**响应示例**:
```json
{
  "not_crawled": 50,      // 未爬取数量
  "crawling": 2,          // 正在爬取数量
  "completed": 100,       // 已完成数量
  "total": 152,           // 总数
  "total_comments": 1500, // 评论总数
  "last_crawl_time": "2026-03-19 10:30:00"
}
```

---

### 2. 获取笔记列表 `/api/notes`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | string | 否 | 关键词筛选（如：美妆、穿搭） |
| status | string | 否 | 状态：`crawling`(爬取中) / `completed`(已完成) |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认10，最大100 |

**响应示例**:
```json
{
  "data": [
    {
      "note_id": "67b176a9000000001703b7ae",
      "title": "2024年最火的口红色号推荐",
      "author_name": "美妆达人小红",
      "publish_time": "2024-03-15",
      "like_count": 12580,
      "keyword": "美妆",
      "is_comment_crawled": 1,  // 1=已完成, 2=爬取中
      "comment_count": 1523,
      "create_time": "2026-03-19T10:30:00",
      "update_time": "2026-03-19T10:30:00"
    }
  ],
  "pagination": {
    "page": 1,
    "page_size": 10,
    "total": 100,
    "total_pages": 10
  }
}
```

## 三、数据状态说明

### `is_comment_crawled` 字段

| 值 | 说明 | 前端展示 |
|----|------|----------|
| 0 | 未爬取 | 不展示 |
| 1 | 已完成 | 正常展示 |
| 2 | 正在爬取 | 显示"爬取中"标识/动画 |

## 五、启动方式

### 启动后端服务
```bash
# 方式1
python start_server.py

# 方式2
uvicorn server.main:app --reload --host 0.0.0.0 --port 8000
```

### 启动前端
直接打开 `web/index.html`，或使用本地服务器

### 爬虫独立运行
```bash
# 运行爬虫脚本（独立于后端服务）
python xhs_guding_page.py
```

## 六、注意事项

1. **CORS已配置**：后端已允许所有来源的跨域请求
2. **状态同步**：爬虫脚本会自动更新数据库状态（0→2→1），前端只需读取即可
3. **实时性**：不需要WebSocket，前端可定时轮询 `/api/stats` 获取最新状态
4. **分页处理**：所有列表接口都支持分页，前端需要处理分页逻辑
